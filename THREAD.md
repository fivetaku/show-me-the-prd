# 스레드 메인글

---

바이브코딩을 시작한 사람들이
가장 어려워하는 건??

**개발 문서 만들기.**

"인보이싱 앱 만들어줘"

이 한 마디로 시작하면
AI가 결제, 세금, 다국어, 보고서까지 다 넣으려 합니다.
결과? 엉망진창.

---

진짜 문제는 코딩이 아닙니다.

**기획이 없는 상태에서 AI한테 코드를 시키는 것.**

개발자들은 코드 짜기 전에
PRD, 데이터 모델, 기술 스펙을 먼저 정리합니다.

근데 바이브코더는 이걸 어떻게 만드는지 모릅니다.
"OAuth"가 뭔지, "ERD"가 뭔지도 모르는데
어떻게 기획 문서를 쓰나요.

---

그래서 만들었습니다.

**Show Me The PRD** 🎤
(쇼미더피알디)

한 문장만 말하면
인터뷰 5-6번으로 기획 문서 4종이 자동 완성됩니다.

```
/show-me-the-prd 사진 정리 앱 만들고 싶어
```

---

**어떻게 작동하나요?**

AI가 면접관처럼 질문합니다.
근데 "OAuth를 선택하시겠습니까?" 같은 질문이 아니라:

"로그인 방식을 골라주세요:

1. 소셜 로그인 (추천)
   구글/카카오로 로그인. 비밀번호 관리 불필요.
   다만 구글/카카오에 의존하게 돼요.

2. 이메일+비밀번호
   자유도 높지만, 비밀번호 찾기 기능도 만들어야 해서
   시간이 더 걸려요."

**모든 선택지에 설명 + 장단점.**
모르겠으면 (추천) 고르면 됩니다.

---

**5-6번 질문에 답하면 나오는 문서 4종:**

📋 **PRD** -- 뭘 만드는지, 누가 쓰는지, 뭘 안 만드는지
📊 **데이터 모델** -- 앱이 다루는 핵심 데이터 구조
📅 **Phase 분리** -- 한번에 다 만들지 않고 단계별로
⚙️ **프로젝트 스펙** -- AI한테 "이건 절대 하지 마" 규칙

특히 4번 프로젝트 스펙의 **"절대 하지 마" 목록**이 핵심.
이거 없으면 AI가 API 키를 코드에 박아넣거나
DB 구조를 맘대로 바꿔버립니다.

---

**기존 도구랑 뭐가 다른가요?**

ChatPRD → PM용. 개발자 용어 난무. SaaS.
Spec Kit → 이미 기획이 있는 사람용. 입력이 필요.
Deep Trilogy → 역시 기획 완료 후 단계.

Show Me The PRD → **기획 자체가 없는 사람용.**
한 문장에서 시작해서, 위 도구들에 넣을 입력 문서를 만들어줍니다.

```
아이디어 → [Show Me The PRD] → 4종 문서 → [Spec Kit 등] → 코드
          ~~이 구간이 비어있었음~~
```

---

**실시간 웹 리서치도 합니다.**

"사진 앱 만들고 싶어"라고 하면
비슷한 앱을 실시간으로 검색해서

"조사해봤는데, 이런 기능들이 일반적이에요:
✅ 사진 업로드 -- 간단
⚠️ 얼굴 인식 분류 -- 외부 API 필요, 비용 발생
🔴 실시간 공유 -- 많이 복잡, 첫 버전에선 빼는 게 좋아요"

하드코딩된 추천이 아니라 **근거 기반 추천**.

---

**"진짜 제품" 지향**

로컬에서 목업 데이터로 돌아가는 데모가 아닙니다.
각 Phase마다 체크리스트:

✅ 실제 DB 연결 (목업 X)
✅ 실제 인증 (하드코딩 비밀번호 X)
✅ 실제 서버 배포 (localhost X)
✅ 다른 사람이 접속 가능
